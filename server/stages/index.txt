Stage 0 — Intake & Snapshot

Purpose: Freeze the source of truth.

Input
    GitHub URL or local path
Actions
    Clone repo
    Lock dependency versions
    Read package.json
    Hash files (for caching / retries)

Stage 1 — Static Analysis

Purpose: Understand what the project is, not how to convert it.

Actions
    Detect framework (React + Vite)
    Detect router
    Extract route map
    Detect entry poin
    Detect env vars
    Detect global CSS
    Detect assets

Stage 2 — Migration Planning (AI-assisted)

Purpose: Decide how to convert, not perform conversion.

Actions

Feed analysis JSON to AI
Ask for:
    route → file mapping
    layout strategy
    known risks
    assumptions

Stage 3 — Target Scaffolding

Purpose: Create a safe, predictable destination.

Actions
    Generate Next.js App Router project
    Create /app structure
    Create layout.tsx
    Create empty page.tsx shells
    Copy static assets

Stage 4 — Atomic File Conversion (AI core stage)

Purpose: Convert files one at a time, safely.

Unit of work
    One source file → one target file

Actions
    Provide:
        source file
        target path
        migration plan
        hard constraints
    AI rewrites file content only

Stage 5 — Cross-file Reconciliation

Purpose: Fix mismatches introduced by isolation.

Actions
    Normalize imports
    Fix relative paths
    Align renamed hooks (useNavigate → useRouter)
    Update env usage

Stage 6 — Build & Type Validation

Purpose: Make it actually run.

Actions
    next build
    tsc
    ESLint
Capture:
    Type errors
    Module resolution errors
    Runtime warnings

Stage 7 — Error Repair Loop (AI repair mode)

Purpose: Patch errors with minimal changes.

Actions
For each error:

    Provide:
        error message
        related file
    AI instructed:
        Fix only what’s required to resolve this error.


This loop continues until:
    build passes
    or error budget exceeded

Stage 8 — Finalization

Purpose: Produce a clean, explainable result.

Actions
    Remove unused deps
    Generate README
    Output conversion report

Optional Stage 9 — Confidence Scoring (nice-to-have)

Purpose: Tell user how safe the result is.

Signals
    number of AI fixes
    build retries
    skipped files
